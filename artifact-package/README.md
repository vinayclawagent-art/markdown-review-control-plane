Source: [[Markdown Collaboration Layer for AI-Native Teams]]

## Why this matters

## Product frame
```text
Product: Markdown Review Control Plane
Customer: AI-native teams keeping specs, skills, prompts, and docs in markdown repos
Pain: review state is split between Google Docs, GitHub PRs, Notion, and local files
Promise: Docs-like review without losing markdown as the agent-readable source of truth
MVP: local static review board + JSON comments + CLI export/import
Monetization: OSS wedge with paid migration/service package for teams moving AI specs into markdown
Validation test: use on one X-Intel artifact package README and measure whether comments become merged edits in one day
```
